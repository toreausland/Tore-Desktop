2. Hva er Claude Code?

Claude Code er en lokal AI-agent fra Anthropic som kjører direkte på Tores maskin. Den har lesetilgang til hele filsystemet (inkludert OneDrive/VDR), kan opprette og redigere Markdown-filer (.md), og kan analysere binære filer (.docx/.pdf) via PowerShell-ekstraksjon.

Begrensninger: Claude Code kan ikke generere binære Word-filer (.docx). Alle nye dokumenter er skrevet som .md (Markdown), som kan konverteres til Word via Claude.ai chat, Pandoc, eller manuell kopi.

3.3 §2.2 Claims Discipline

Et kommunikasjonsrammeverk implementert i alle skills og nye dokumenter:

§2.2-reglene i kortform:
1. Skill forsiktig projeksjonsspråk — kvalifiser alle fremtidsrettede utsagn
2. Atskill observert data fra projeksjoner — aldri blend fakta og mål
3. Oppgi kilder — REV6, bransjedata, offentlige kilder
4. Unngå absolutte påstander — «vil», «garanterer», «sikrer»
5. Anticipér motspørsmål — inkluder begrensninger og forutsetninger

3.4 Andrés markedsdokument — CNC Benchmark

André leverte et utkast til konkurranseanalyse/CNC-benchmark. Dette ble implementert som 03_Competitive_CNC_Benchmark.md i VDR 03.02, oppdatert for §2.2-disiplin, og integrert som referansedokument på tvers av VDR-en. Dokumentet:

  • Benchmarker 7 norske HMLV-verksteder (Rogaland Maskinering, Lilaas, Tamek, Uvdal, Aarbakke, TP-Products, Årdal Maskinering)
  • Dokumenterer observert utnyttelse: 20–47 % (gjennomsnitt ~30 %)
  • Forklarer strukturelle årsaker til lav utnyttelse i brownfield-anlegg
  • Brukes nå som kildedokument i Sensitivitetsanalyse, Prisingsmodell, Risikoregister og GTM-strategi

6. Kritisk feiloppdagelse og korrigering

Tore oppdaget at Finansieringsplanen avvek fra REV6. Serie A-bankgjeld sto som 29,5 MNOK — korrekt verdi er 105 MNOK (15 CNC × 7,0 MNOK). Feilen skyldtes at Claude Code ikke hadde direkte lesetilgang til REV6 .docx-filen da dokumentene ble generert, og baserte seg på fragmentert kontekst fra andre dokumenter.

6.1 Rotårsak

REV6 er en .docx-fil (binær). Claude Code kunne ikke lese den direkte ved første generering. Tallene ble hentet fra sammendrag i andre dokumenter, og Seed-logikken (5 MNOK gjeld/CNC) ble feilaktig kopiert til Serie A (korrekt: 7 MNOK gjeld/CNC).

11. Anbefalinger

  1. Prioritér §2.2-korrigeringer i Executive Summary og Investment Teaser — dette er det investorer ser først. 8 kritiske endringer tar ~30 minutter.
  2. Konverter de 8 nye .md-dokumentene til Word — Tore har startet dette via Claude.ai chat.
  3. Verifiser alltid mot REV6 Section 8 — 17 checkpoint-verdier som er den endelige sannheten.
  4. Bruk Claude Code for fremtidig dokumentgenerering — men les REV6 direkte først (via PowerShell-ekstraksjon).
